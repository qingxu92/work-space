# PRD Creator Skill

基于用户需求描述，通过结构化问卷澄清需求细节，生成专业PRD文档。

## 触发场景

当用户：
- 明确要求生成 PRD
- 描述页面功能需求
- 说"帮我写一个XXX页面的PRD"
- 提供功能清单要求生成需求文档

## 核心特点

| 特点 | 说明 |
|------|------|
| **通用性强** | 支持多种产品类型，不限定某一类型 |
| **专业输出** | 输出产物专业，不泛泛而谈 |
| **用户可选** | 用户选择产品类型和页面类型 |
| **需求澄清** | 结构化问卷确保完全理解用户意图 |

## 支持的产品类型

| 产品类型 | 说明 |
|---------|------|
| 电商类 | 电商平台、商城、O2O等 |
| 内容类 | 资讯、社区、博客、视频等 |
| 工具类 | 效率工具、管理系统等 |
| 金融类 | 支付、理财、保险等 |
| 社交类 | IM、社交网络、论坛等 |
| 教育类 | 在线教育、学习平台等 |

## 工作流程

1. **触发**：用户请求生成PRD
2. **选择产品类型**：电商/内容/工具/金融/社交/教育
3. **选择页面类型**：根据产品类型动态展示可选页面类型
4. **结构化问卷**：逐项澄清需求细节
5. **确认理解**：向用户确认需求理解是否正确
6. **生成PRD**：套用专业模板生成PRD文档
7. **交付**：输出PRD文件，询问是否需要修改

## 结构化问卷维度

| 维度 | 内容 |
|------|------|
| 页面定位 | 页面名称、核心功能、目标用户、使用场景 |
| 功能模块 | 功能清单、模块划分、功能优先级 |
| 字段定义 | 字段名称、类型、验证规则、默认值 |
| 交互规则 | 操作流程、状态变化、联动逻辑 |
| 权限规划 | 权限点、操作范围、数据范围 |
| 异常场景 | 网络异常、数据异常、权限异常 |
| 验收标准 | 核心验收点、边界条件 |

## 输出要求

生成的PRD文档应包含：
1. 引言（背景、功能清单）
2. 页面定位与用户角色
3. 功能模块详述
4. 交互设计
5. 异常处理
6. 验收标准

## 使用示例

```
用户：帮我写一个商品详情页的PRD
助手：请选择产品类型：[电商/内容/工具/金融/社交/教育]
用户：电商
助手：请选择页面类型：[首页/商品列表/商品详情/购物车/订单...]
用户：商品详情
助手：[开始结构化问卷...]
```
