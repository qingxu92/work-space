# PRD Generator Skill

基于页面代码、spec.md、content.md、截图等资料，自动识别页面类型，套用对应PRD模板，生成标准化的产品需求文档。

## 触发场景

当用户：
- 明确要求生成 PRD
- 要把已实现页面转成 PRD
- 要根据 spec.md 生成正式 PRD

## 输入资料

优先读取目标页面目录下的：
- `index.tsx` - 页面主组件代码
- `spec.md` - 规格说明文档
- `content.md` - 内容配置文档
- `screenshot.png` - 页面截图

如果 `spec.md` 或截图不存在，就基于页面实现继续生成，不要中断。

## 页面类型识别

必须识别以下 5 类之一：

| 类型 | 识别特征 |
|------|---------|
| 数据看板类 | 包含图表、统计卡片、数据可视化组件 |
| 后台列表类 | 包含表格、筛选条件、分页、批量操作 |
| 后台表单编辑类 | 包含表单输入、提交按钮、字段验证 |
| 后台表单预览类 | 包含只读字段展示、无编辑操作 |
| 用户前台类 | 面向C端用户、包含商品展示、购物流程等 |

## 模板选择

根据页面类型选择对应模板：
- `data-prd-template.md` - 数据看板类
- `backend-list-prd-template.md` - 后台列表类
- `backend-form-edit-prd-template.md` - 后台表单编辑类
- `backend-form-preview-prd-template.md` - 后台表单预览类
- `user-frontend-prd-template.md` - 用户前台类

## 工作流程

1. **接收请求**：确认目标页面和需求范围
2. **读取资料**：读取 index.tsx、spec.md、content.md、screenshot.png
3. **识别类型**：根据页面结构判断页面类型
4. **选择模板**：按页面类型选择对应 PRD 模板
5. **提取信息**：提取功能模块、字段、按钮、交互、状态、权限、异常场景和验收点
6. **生成PRD**：按模板生成 PRD 内容
7. **保存文件**：保存到 `src/pages/<page-name>/prd.md`
8. **核对验证**：核对 PRD 与页面实现、spec.md、用户范围是否一致
9. **修正交付**：如发现遗漏或不一致，修正后再交付

## 异常处理

- 页面类型无法识别时，先向用户确认
- 页面很复杂时，拆成多个模块分别描述，但最终仍输出一个完整的 `prd.md`
- 不要编造页面中不存在的功能

## 输出要求

生成的 PRD 文件应包含：
1. 引言（背景、功能清单）
2. 页面菜单与权限规划
3. 各模块需求说明
4. 交互设计
5. 异常处理说明

## 使用示例

```
用户：帮我生成用户列表页的PRD
助手：[读取页面资料 → 识别为后台列表类 → 选择模板 → 生成PRD]

用户：把这个页面转成PRD文档
助手：[读取当前页面 → 识别类型 → 生成PRD]
```
