# Eagle 素材管理项目规则

## 一、项目说明

**项目路径**：`d:\Trae 库\需求分析`
**主要工作**：管理 Eagle 软件中的素材库，包括素材分类、标签管理、搜索整理等

---

## 二、工作模式

### 2.1 默认模式：变更清单确认

- 执行任何修改操作前，先输出变更清单
- 变更清单应包含：修改内容、影响范围、风险提示
- 等待用户确认后再执行实际操作
- 仅在用户明确同意后才进行文件写入或修改

### 2.2 特殊指令优先

当用户明确使用以下指令时，按对应模式执行：

| 指令 | 模式 | 说明 |
|------|------|------|
| /plan | 计划模式 | 先制定计划，用户确认后再执行 |
| /spec | 规格模式 | 先编写详细规格说明书，用户确认后再执行 |

## 三、任务执行流程

### 3.1 任务规划

1. 使用 `TodoWrite` 工具创建任务清单
2. 按优先级排序：P0 > P1 > P2
3. 标记任务状态：pending / in_progress / completed

### 3.2 任务执行原则

1. **一次性原则**：同一任务尽量一次完成，避免反复
2. **闭环原则**：每个任务必须有结果或明确的下一步
3. **记录原则**：重要决策和发现及时记录到文档中

### 3.3 任务完成标准

- 素材操作已完成且可验证
- 无数据丢失风险
- 结果已保存/输出

## 四、文件操作规范

### 4.1 文件编码规范

| 要求 | 说明 |
|------|------|
| 编码格式 | UTF-8 无 BOM |
| 命名规范 | 使用中文命名，简洁明了 |
| 路径规范 | 使用绝对路径 |

### 4.2 文件修改流程

1. 使用 `Read` 工具完整读取文件
2. 在内存中修改内容
3. 使用 `Write` 工具完整重写文件
4. 禁止追加式写入

### 4.3 文件创建规范

1. 先确认文件是否已存在
2. 已存在文件必须先 Read 再 Write
3. 新文件创建后告知用户文件路径

## 六、Eagle 素材管理专项规则

### 6.1 素材查询原则

1. **明确需求**：先理解用户想要查找的素材类型、关键词、标签等
2. **精准搜索**：使用 `ai_search_by_text` 或 `item_query` 进行搜索
3. **结果展示**：按相关性排序，展示素材数量、缩略图、标签等信息

### 6.2 素材管理操作

| 操作 | 说明 | 注意事项 |
|------|------|---------|
| 添加标签 | 为素材添加或修改标签 | 标签统一使用中文，避免重复 |
| 创建文件夹 | 创建新的素材分类文件夹 | 按主题或项目命名 |
| 移动素材 | 将素材移动到指定文件夹 | 确认目标文件夹存在 |
| 删除素材 | 将素材移到回收站 | 提醒用户操作不可逆 |
| 批量操作 | 对多个素材进行批量处理 | 谨慎操作，确认范围 |

### 6.3 素材分类规范

| 分类层级 | 说明 | 示例 |
|---------|------|------|
| 一级分类 | 按大类别划分 | 设计素材、产品图片、文档资料 |
| 二级分类 | 按子类别划分 | UI组件、图标、插画 |
| 标签 | 细化描述素材属性 | 移动端、红色、按钮 |

### 6.4 标签命名规范

- 使用中文标签，简洁明了
- 避免重复标签（如"按钮"和"button"）
- 标签层级：主标签+子标签（如"UI组件-按钮"）
- 定期清理无效标签

### 6.5 风险提示

发现以下情况必须主动提示用户：

- 批量操作影响范围大时
- 删除操作不可逆时
- 素材数量过多可能导致性能问题时
- 标签冲突或重复时

## 七、日志记录规范

### 7.1 日志目录

**日志路径**：`d:\Trae 库\需求分析\logs\logs.md`

### 7.2 日志记录要求

- 每次修改或更新文件后，必须在日志文件中添加一条记录
- 日志记录应包含：时间、操作人、操作类型、操作对象、描述
- 日志按日期分组，最新记录放在最上方

### 7.3 日志格式

| 字段 | 说明 | 示例 |
|------|------|------|
| 时间 | YYYY-MM-DD HH:MM:SS | 2026-05-07 09:00:00 |
| 操作人 | 执行操作的用户或系统 | 用户/系统 |
| 操作类型 | 新增/修改/删除/查询 | 修改 |
| 操作对象 | 被操作的文件或素材 | rules.md |
| 描述 | 操作内容简述 | 更新工作模式为变更清单确认 |

### 7.4 日志记录流程

1. 完成文件修改或操作后
2. 读取日志文件
3. 在对应日期下添加新记录
4. 保存日志文件

## 八、沟通规范

### 8.1 主动告知事项

- 任务开始时告知预计步骤
- 任务完成时总结交付物
- 发现问题时说明风险和建议

### 8.2 询问时机

以下情况应主动询问用户：

- 批量操作影响范围大
- 删除或移动操作不可逆
- 用户未明确说明的模糊需求
- 多个合理方案可选时

### 8.3 任务交接

当任务需要用户确认才能继续时：

1. 明确说明当前状态
2. 列出待确认事项
3. 提供选项或建议
